# ГИМС 2026 — тренажёр

Одностраничный клиентский тренажёр для подготовки к **теоретическому экзамену ГИМС МЧС России**
(судоводитель-любитель), редакция правил **2026** (Приказ МЧС № 636). Работает полностью в браузере,
без бэкенда, хранит прогресс в `localStorage`, разворачивается на GitHub Pages.

> ⚠️ Параметры экзамена (20 вопросов / 40 минут / проходной 80%) и все материалы взяты из приложенных
> учебных файлов. Авторитетный источник — актуальный приказ МЧС России № 636. Сверяйтесь с
> официальными требованиями ГИМС перед экзаменом.

## Возможности

- **Мой план** — дашборд: сводка, быстрый старт, прогресс по разделам, слабые темы.
- **Режим изучения** — 434 карточки справочника (вопрос → ответ), «Знаю / Не знаю», избранное, закладки, перемешивание.
- **Режим экзамена** — 20 вопросов, таймер 40 минут с авто-завершением, вердикт по порогу 80%, полный разбор.
- **Билеты** — 5 готовых билетов × 20 вопросов с разбором (пояснение, мнемоника, плашка «Ловушка»).
- **Марафон** — весь пул вопросов подряд с мгновенной проверкой; позиция сохраняется.
- **Топ ошибок**, **Избранное**, **Закладки**, **Поиск**, **Статистика** (графики Recharts), **Сброс**.
- **Морская тема**: светлая «дневная» по умолчанию (для чтения при дневном свете) + тёмная «глубокое море», переключатель ☀️/🌙. Мобайл-фёрст, **3 ступени размера шрифта** (A / A+ / A++), доступность (ARIA, видимый фокус, контраст WCAG AA, таргеты ≥ 44px), PWA-манифест.

## Стек

Vite + React + TypeScript + Tailwind CSS · Recharts · lucide-react. Маршрутизация — по hash (`#/exam`,
`#/study/1.1`), без react-router. Без сетевых запросов в рантайме (все данные — локальные JSON из бандла).

## Структура

```
gims-2026/
  source/                 исходные PDF (+ промежуточные .txt от pdftotext, в .gitignore)
  scripts/build-data.js   парсер источников → JSON
  public/
    data/                 tickets.json, reference.json, rules.json (закоммичены)
    manifest.webmanifest, icon.svg
  src/
    main.tsx, App.tsx     каркас, боковое меню, роутинг по hash
    config.ts             examConfig (20 / 40 мин / 80%)
    router.tsx, nav.ts
    data/                 типы и загрузчики JSON
    store/                localStorage (ключ gims2026:v1, версионирование + миграция)
    ui/                   prefs (шрифт), quiz, stats, util
    components/           QuestionCard, Timer, ProgressBar, ResultScreen, SearchBar, StatChart, Sidebar, …
    screens/              по экрану на раздел (13 шт.)
  vite.config.ts          base: './'  (важно для GitHub Pages)
```

## Команды

Проект использует **Bun** (есть `bun.lock`). Команды `npm` тоже работают.

```bash
# установка зависимостей
bun install            # или: npm install

# локальный запуск (dev-сервер)
bun run dev            # http://localhost:5173

# продакшн-сборка (tsc --noEmit + vite build → dist/)
bun run build

# просмотр собранного билда (проверка base: './')
bun run preview

# проверка типов
bun run typecheck
```

### Пересборка данных из PDF

Требуется **pdftotext** (Poppler/Xpdf) в `PATH`. Скрипт сам конвертирует `source/*.pdf` → `.txt`
(режимы `-table` для справочника, `-layout` для тестов и правил) и собирает три JSON в `public/data/`.

```bash
bun run data           # = bun scripts/build-data.js
```

В конце печатается отчёт: число билетов/вопросов, карточек по разделам, глав правил и счётчик
`needsReview` (см. ниже).

## Данные и `needsReview`

Источники парсятся «как есть», **без выдумывания**. Если в источнике текст обрезан/нечитаем — элемент
помечается `needsReview: true`, пробел не заполняется.

| Файл | Содержимое | Итог |
|---|---|---|
| `tickets.json` | 5 билетов × 20 = **100** тестовых вопросов | чисто, `needsReview: 0` |
| `rules.json` | **7** глав A0–A6 (заголовки, абзацы, таблицы, плашки 2026, мнемоники) | A5 — таблица штрафов |
| `reference.json` | **434** карточки, 14 разделов в 5 блоках | см. примечание |

> **Справочник** — это *сжатая* (compressed) редакция PDF: у ~358 из 434 карточек **начало вопроса
> физически обрезано в исходнике**, а у некоторых ответ «затекает» из соседней строки (особенности
> вёрстки). Ответы при этом в большинстве корректны. Такие карточки помечены `needsReview: true` и
> в интерфейсе получают плашку «требует проверки». Для идеального качества пересоберите данные из
> полной (не сжатой) версии справочника.

## Деплой на GitHub Pages

`vite.config.ts` уже содержит `base: './'` — ассеты подхватываются в подпапке репозитория.
Данные `public/data/*.json` **закоммичены**, поэтому в CI `pdftotext` не нужен.

### Вариант A — GitHub Actions (рекомендуется)

В репозитории есть `.github/workflows/deploy.yml`. Включите Pages:
**Settings → Pages → Build and deployment → Source: GitHub Actions**. После пуша в `main` сайт
соберётся и опубликуется автоматически. Репозиторий должен быть **Public** (или Pages включён по тарифу).

### Вариант B — ручная публикация через ветку `gh-pages`

```bash
bun run build
bun run deploy         # пакет gh-pages публикует dist/ в ветку gh-pages
```
Затем: **Settings → Pages → Source: Deploy from a branch → `gh-pages` / root**.

## Хранение прогресса

Единый ключ `localStorage` — `gims2026:v1` (версионируется, при отсутствии/ошибке безопасно создаётся
заново). Хранит историю попыток, статистику карточек и вопросов, избранное, закладки, позицию марафона,
дату визита и серию дней. Настройки оформления — отдельные ключи: размер шрифта `gims2026:font`
(`normal` / `large` / `xlarge`) и тема `gims2026:theme` (`light` / `dark`).
